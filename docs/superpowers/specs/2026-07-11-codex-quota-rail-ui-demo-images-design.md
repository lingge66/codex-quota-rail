# Codex Quota Rail UI 演示效果图设计规格

- 状态：用户已确认视觉方向 A（真实桌面演示）
- 日期：2026-07-11
- 用途：GitHub README、X（Twitter）与 QQ 社群宣传
- 交付：4 张 PNG 位图

## 1. 设计目标

制作一套可信、克制、能直接说明产品价值的 UI 演示图。画面以真实 Codex 桌面工作界面为主体，重点展示额度轨贴在 Codex 窗口顶部外侧、额度颜色随剩余比例变化、点击详情和最大化紧凑模式。演示图不得把尚未实现的能力画成现有功能。

视觉方向采用已确认的 A：真实桌面演示。画面可以使用轻微景深、阴影和绿色环境光提升质感，但不得牺牲 UI 可读性，也不得把额度轨画成独立悬浮工具栏。

## 2. 成品清单

### 2.1 普通窗口主图

- 文件：`docs/assets/demo/codex-quota-rail-normal-wide.png`
- 画幅：横版，约 3:2，适合 GitHub README。
- 场景：Windows 深色 Codex 普通窗口，额度轨位于窗口顶部外侧并与窗口同宽。
- 状态：`5 小时 可用 90%`、`本周 可用 68%`，均为绿色。
- 重点：22px 双额度轨清晰可见，不遮挡菜单栏和 Codex 内容。

### 2.2 点击详情图

- 文件：`docs/assets/demo/codex-quota-rail-details-wide.png`
- 画幅：横版，约 3:2，适合 GitHub 功能说明。
- 场景：Windows 深色 Codex 普通窗口，用户刚点击 22px 额度轨，详情卡从额度轨下方展开。
- 状态：`5 小时 可用 78%` 为绿色，`本周 可用 42%` 为黄色；显示对应重置时间和最近更新时间。
- 重点：详情卡宽度克制，不获得键盘焦点，不长期遮住顶部功能；静态图只表达“点击后展开”的瞬时状态。

### 2.3 低额度预警图

- 文件：`docs/assets/demo/codex-quota-rail-low-quota-square.png`
- 画幅：1:1 方图，适合 X（Twitter）与 QQ 社群。
- 场景：Windows 深色 Codex 普通窗口局部，额度轨仍在窗口顶部外侧。
- 状态：短周期额度为 `可用 18%`，轨道缩短并变为红色；长周期额度保持绿色或黄色以形成对比。
- 重点：明确表达颜色代表剩余可用额度，而非已消耗额度；不使用夸张警报弹窗。

### 2.4 最大化紧凑图

- 文件：`docs/assets/demo/codex-quota-rail-compact-square.png`
- 画幅：1:1 方图，适合 X（Twitter）与 QQ 社群。
- 场景：Windows 深色 Codex 最大化窗口。
- 状态：额度轨自动降级为窗口最顶部约 4px 的双额度紧凑线，不显示文字和详情卡。
- 重点：紧凑线不遮挡标题栏、菜单或内容，能看出绿色与黄色两个额度状态。

## 3. 统一视觉规则

- 深色 Codex 桌面界面作为主体，保留真实 Windows 窗口结构和中文 Codex 导航语境。
- 额度轨必须附着在 Codex 窗口上：普通窗口位于顶部外侧，最大化时位于顶部内侧并缩为 4px。
- 普通轨包含 `CODEX`、两个可用额度标签、百分比和进度；详情卡标题为 `Codex 可用额度`。
- 颜色遵循现有产品规则：51%–100% 绿色，21%–50% 黄至橙色，1%–20% 红色。
- 轨道长度表达剩余可用比例：90% 应接近满轨，18% 应只保留短段。
- 使用领哥品牌气质，但不重绘或变形领哥 LOGO；如需品牌识别，只使用现有 LOGO 原图作为小面积角标。
- 允许轻微暗绿色环境光、柔和窗口阴影和清晰的屏幕层次；禁止霓虹过曝、悬浮 3D 轨道或科幻 HUD。
- 不添加 OpenAI 官方标识、官方合作暗示、虚构系统按钮、虚构额度窗口或无法工作的交互。
- 不在演示图内加入长段宣传文案，避免文字失真并保持 GitHub 阅读体验。

## 4. 输入图角色

- 真实 Windows 截图：界面结构与额度轨交互参考，不要求逐像素复刻截图中的临时项目内容。
- 现有概念图：只参考深色、克制绿色边框和外侧贴边关系，不沿用“独立悬浮条”的错误观感。
- 领哥 LOGO：品牌参考；不得修改字形、比例或黑白结构。

## 5. 生成与验收方式

- 使用内置图片生成能力，一张图一个独立提示词，避免不同场景互相污染。
- 生成后逐张检查额度条位置、颜色、比例、中文文字、窗口层级和画幅。
- 发现文字或 UI 结构错误时，只针对单一问题迭代，不用错误图充当正式成品。
- 最终成品复制到 `docs/assets/demo/`，不得只保留在临时生成目录。
- GitHub 横图和社群方图都应在缩略图尺寸下仍能辨认额度轨颜色与位置。
