# Codex Quota Rail 设计规格

- 状态：用户已确认规格
- 日期：2026-07-10
- 首发平台：Windows x64
- 工作名称：Codex Quota Rail（额度边缘轨）

## 1. 结论

Codex Quota Rail 是一个非 OpenAI 官方的 Windows 开源常驻工具。它不修改 Codex 安装文件，而是在 Codex 主窗口顶部外侧附着一条 22px 高的额度边缘轨，通过官方 Codex App Server 读取账户额度，并统一转换为“可用额度”展示。

普通窗口下，边缘轨位于窗口顶部外侧，不遮挡 Codex 内容；Codex 最大化时，边缘轨自动降级为标题栏顶部约 4px 的双色进度线。工具随 Codex 移动、缩放、最小化和跨屏移动，Codex 失焦后仍显示但降低透明度。

首版采用 C#、.NET 8、WPF 和 Win32 实现，优先把 Windows 窗口附着、视觉效果、安装体验和异常恢复做到稳定。数据层、窗口层与渲染层相互隔离，为后续 macOS 外壳和“Codex 宠物额度皮肤”预留接口。

## 2. 目标

- 用户不再进入账号菜单和使用量页面即可查看 Codex 可用额度。
- 主额度与次额度始终可同时查看；只有一个额度窗口时自动使用单轨。
- 展示口径统一为“可用额度”，100% 代表额度充足，0% 代表额度耗尽。
- 视觉上像 Codex 窗口的一部分，技术上保持独立，避免 Codex 更新破坏工具。
- 默认不抢键盘焦点、不进入任务栏、不持续播放干扰动画。
- 开源发布，提供安装包、便携版、中英文文档和可验证的构建产物。
- 不读取聊天内容、不持久化账号令牌、默认零遥测。

## 3. 非目标

- 首版不修改、注入、解包或重新打包 Codex 桌面应用。
- 首版不通过网页抓取、屏幕识别或模拟点击获取额度。
- 首版不支持 macOS 或 Linux，但核心接口不得阻碍后续移植。
- 首版不静默自动更新，只提供手动“检查更新”。
- 首版不实现宠物皮肤，只保留可替换渲染接口和状态模型。
- 首版不提供账户登录界面；用户在 Codex 中完成登录。
- 首版不发送匿名统计、崩溃报告或行为分析。

## 4. 用户体验

### 4.1 默认形态

- 高度：22px。
- 位置：Codex 主窗口顶部外侧，水平宽度跟随 Codex。
- 内容：品牌标识、短周期可用额度轨、长周期可用额度轨、最近重置倒计时。
- 轨道语义：轨道长度代表剩余可用比例，而不是已用比例。
- 文案示例：`5 小时 可用 32%`、`本周 可用 59%`。
- 窄窗口降级顺序：隐藏完整重置文案、隐藏窗口名称补充词、保留百分比和两条进度轨。
- 单额度窗口：单轨居中显示，不渲染空轨或假 0%。
- 无限额度：显示“无限”，不伪造 100% 进度。

### 4.2 颜色与动效

可用额度颜色由高到低连续变化，同时保留文字状态，不能只依赖颜色表达：

- 51%–100%：绿色区间。
- 21%–50%：黄色逐渐过渡到橙色。
- 1%–20%：红色区间。
- 0%：红色、柔和呼吸点和低速跑马灯，明确显示重置时间。

动效规则：

- 正常状态不循环播放动画。
- 跨入 50% 或 20% 阈值时播放一次约 1.2 秒的流光提示。
- 只有额度达到 0% 才持续使用低速跑马灯；额度恢复后立即停止。
- 用户启用“减少动画”后，所有流光、呼吸和跑马灯改为静态状态变化。

### 4.3 焦点、悬停和窗口状态

- Codex 为前台窗口时，边缘轨使用完整不透明度。
- Codex 可见但未聚焦时，边缘轨在约 180ms 内降至 52% 不透明度。
- Codex 重新聚焦时恢复完整不透明度。
- 鼠标悬停约 250ms 后展示详情卡，包含每个额度窗口的可用比例、重置时间和最近更新时间。
- 详情卡只处理鼠标输入，不获取键盘焦点。
- Codex 最小化时边缘轨同步隐藏，恢复时同步出现。
- Codex 最大化时，切换为标题栏顶部约 4px 的双色进度线；悬停仍可展示详情卡。
- Codex 退出最大化后恢复顶部外侧 22px 双轨。
- 多个 Codex 窗口存在时，首版只跟随最近激活的主窗口。
- 跨显示器移动时按目标显示器 DPI 重新计算位置和尺寸，避免明显跳变。
- 如果另一个普通窗口覆盖 Codex，边缘轨与 Codex 维持相同的 Z 序关系，不强行覆盖其他应用。

### 4.4 托盘与启动

首次安装明确展示“开机时启动”选项，默认勾选。用户可在托盘菜单中：

- 查看连接状态与最近更新时间。
- 手动刷新额度。
- 暂停或恢复窗口跟随。
- 选择自动、深色或浅色主题。
- 启用或关闭减少动画。
- 启用或关闭开机自启。
- 打开日志目录和故障排查页面。
- 手动检查更新。
- 退出程序。

程序必须为单实例。重复启动时激活托盘菜单或显示简短状态提示，不创建第二条边缘轨。

## 5. 技术架构

### 5.1 技术选择

- 运行时：.NET 8。
- UI：WPF。
- Windows 集成：P/Invoke、WinEvent Hook、DWM/窗口定位 API。
- 额度数据：Codex App Server 的 JSONL JSON-RPC 协议。
- 本地设置：`%LocalAppData%/CodexQuotaRail/` 下的版本化 JSON 设置。
- 日志：结构化本地日志，默认不记录原始协议响应和账户标识。

选择 WPF 的原因是透明无边框窗口、托盘、Win32 窗口事件、多显示器和 DPI 处理成熟。Rust/Tauri 保留为未来跨平台评估方案，Electron 因体积和常驻内存成本不用于首版。

### 5.2 组件边界

#### `CodexWindowTracker`

职责：发现 Codex 主窗口并发布窗口快照。

输出包含窗口句柄、边界、显示器、DPI、焦点、可见、最小化和最大化状态。优先使用 WinEvent 事件驱动更新，仅使用低频校准纠正丢失事件。

#### `OverlayWindowController`

职责：根据窗口快照控制边缘轨的位置、大小、透明度、可见性与 Z 序。

它不得改变 Codex 窗口尺寸，不得把边缘轨设为跨应用永久置顶，也不得获取键盘焦点。

#### `CodexAppServerResolver`

职责：发现系统已安装的官方 Codex App Server 能力。

发现顺序：

1. PATH 中可用的官方 `codex` 命令。
2. 已安装 Codex 桌面组件公开可执行的 App Server 入口。
3. 若均不可用，显示官方安装或更新指引。

首版不下载或捆绑来历不明的二进制文件，也不使用网页抓取作为后备方案。

#### `RateLimitClient`

职责：启动或连接 App Server，完成初始化，读取额度快照并监听额度变化。

使用的方法：

- `account/read`：确认账户与认证状态。
- `account/rateLimits/read`：读取完整额度快照。
- `account/rateLimits/updated`：接收额度变化提示；收到后重新读取完整快照。

协议依据：[Codex App Server 官方文档](https://developers.openai.com/codex/app-server)。

#### `QuotaNormalizer`

职责：把 App Server 数据转换为稳定、可测试的应用领域模型。

核心换算：

```text
availablePercent = clamp(100 - usedPercent, 0, 100)
```

接口缺少 `usedPercent` 时不得默认 100%。`unlimited` 为真时输出无限状态；次额度为空时输出单窗口状态。

#### `QuotaStateStore`

职责：保存当前有效快照、最近更新时间、连接状态和错误状态，并向渲染器发布不可变视图模型。

#### `IQuotaRenderer`

职责：把统一的额度视图模型渲染成具体视觉形态。

首版实现 `RailQuotaRenderer`。接口不得暴露 App Server 原始 JSON 或 Win32 句柄，确保二期可新增 `PetQuotaRenderer` 而不改动数据读取与窗口跟随。

#### `TrayApplicationController`

职责：管理单实例、开机自启、托盘菜单、设置、日志、更新检查和程序退出。

### 5.3 数据流

1. 程序启动并建立单实例。
2. `CodexWindowTracker` 发现可见的 Codex 主窗口。
3. `CodexAppServerResolver` 定位官方 App Server 入口。
4. `RateLimitClient` 初始化协议并确认认证状态。
5. 调用 `account/rateLimits/read` 获取完整额度快照。
6. `QuotaNormalizer` 将已用比例换算为可用比例并校验字段。
7. `QuotaStateStore` 发布当前状态。
8. `RailQuotaRenderer` 渲染双轨或单轨。
9. 收到 `account/rateLimits/updated` 后重新读取完整快照。
10. 没有通知时每 60 秒进行一次轻量校准；系统唤醒、网络恢复或 Codex 重启后立即刷新。

## 6. 异常与恢复

### Codex 未启动

边缘轨隐藏，托盘显示“等待 Codex”。检测到 Codex 后自动附着，无需重启本工具。

### 用户未登录

显示“需要先在 Codex 中登录 ChatGPT”。本工具不收集账号密码；用户操作只负责唤起 Codex。

### App Server 暂时断开

保留最后一次有效额度并显示“几分钟前更新”。重连间隔按 2、5、15、30、60 秒退避，成功后重置退避状态。

### Codex 版本不支持接口

显示“当前 Codex 不支持额度接口”和官方更新指引。不启用网页抓取、UI 自动化或估算额度。

### 响应无效或字段缺失

记录脱敏诊断事件，界面显示“额度暂不可用”。不得显示假 100%、假绿色或空白无解释状态。

### 系统休眠与网络变化

暂停无意义的重试；系统恢复或网络重新可用后立即建立新连接并刷新完整快照。

## 7. 隐私与安全

- App Server 使用 Codex 已有登录状态；本工具不读取或持久化访问令牌。
- 不读取任务、聊天内容、项目文件或剪贴板。
- 默认零遥测。
- 原始协议响应不进入普通日志。
- 日志对用户目录、账户标识、进程参数和可能的令牌字段进行脱敏。
- 更新检查只由用户从托盘主动触发。
- 发布包提供 SHA-256 校验值、软件物料清单和第三方许可证清单。
- README 明确标注“非 OpenAI 官方产品”。

## 8. 性能目标

- 后台空闲 CPU 接近 0%，不能依赖高频窗口轮询。
- 常驻内存目标低于 80MB。
- 冷启动到托盘可用目标低于 2 秒。
- Codex 移动或缩放时边缘轨无明显抖动或拖影。
- 收到额度变化后数秒内完成界面更新。
- 长时间断线不能形成高频重试或日志洪泛。

## 9. 测试与验证

### 单元测试

- `usedPercent` 到 `availablePercent` 的边界与异常换算。
- 100%、50%、20%、0% 的颜色与状态映射。
- 单额度、双额度、无限额度和缺字段归一化。
- 重置时间和倒计时格式。
- 退避重连状态机。
- 设置迁移与默认值。

### 协议集成测试

使用模拟 JSONL App Server 覆盖：

- 初始化和认证状态。
- 正常额度快照。
- 额度变化通知后重新读取。
- 超时、断线、无效 JSON、未知字段和缺少字段。
- 不支持的方法和版本不兼容。
- 进程退出与重启。

### Windows 集成测试

- Codex 启动、退出和重启。
- 移动、缩放、最大化、还原和最小化。
- 前后台切换与其他窗口覆盖。
- 多显示器、负坐标、不同 DPI 和缩放比例。
- Windows 资源管理器重启。
- 电脑休眠、唤醒和网络恢复。
- 重复启动单实例保护。

### 视觉与无障碍测试

- 可用额度 100%、50%、20%、0% 的快照测试。
- 普通、失焦、最大化、断线和悬停详情状态。
- 深色、浅色和高对比度。
- 125%、150%、200% 缩放。
- 减少动画模式。
- 颜色以外必须有百分比、文字和状态标签。

### 手动验收门槛

必须在真实 Windows Codex 窗口上观察到以下行为后才算完成：

- 边缘轨真实附着并跟随窗口移动。
- 最大化时正确切换为 4px 模式。
- 最小化和恢复同步。
- 实际账户额度正确换算为可用额度。
- 断开 App Server 后显示最后更新时间并成功恢复。
- Codex 失焦后透明度降低且不覆盖其他应用。

## 10. 开源与发布

- 许可证：MIT。
- 首发产物：Windows x64 安装包和便携 ZIP。
- 安装器：明确询问是否开机自启，默认勾选。
- CI：GitHub Actions 执行恢复、构建、测试、发布、SHA-256 和软件物料清单生成。
- 文档：中英文 README、安装、隐私、安全、架构、故障排查、贡献指南和变更记录。
- 更新：首版仅支持托盘手动检查，不静默下载或安装。
- 签名：发布流程预留 Authenticode/MSIX 签名；没有受信任证书时必须在发布说明中明确 Windows“未知发布者”风险。
- 版本策略：语义化版本；协议兼容性变化必须记录在变更日志。

## 11. 二期：Codex 宠物额度皮肤

宠物功能作为可选渲染器，不替换首版极简边缘轨，也不进入首版实现范围。

建议状态映射：

- 可用 76%–100%：活跃、精神充足。
- 可用 51%–75%：平静工作。
- 可用 21%–50%：略显疲惫并出现轻量提示。
- 可用 1%–20%：明显疲惫，颜色进入红色区域。
- 可用 0%：休眠或趴下，并显示重置倒计时。

二期实现必须满足：

- 使用 `IQuotaRenderer` 接收同一额度视图模型。
- 支持完全关闭宠物与动画。
- 不能增加新的账号权限或数据读取范围。
- 宠物动画遵守减少动画设置。
- 点击或悬停宠物可展开与边缘轨相同的双额度详情。
- 资源包与代码分离，为社区宠物主题预留贡献方式。

## 12. 验收标准

首版只有同时满足下列条件才能发布：

1. 用户无需进入 Codex 使用量页面即可持续看到真实可用额度。
2. 普通、最大化、最小化、失焦和跨屏状态均符合本规格。
3. 100% 为绿色满轨，剩余额度下降时轨道缩短并过渡到红色，0% 才启用跑马灯。
4. 断线、未登录、旧版本和缺字段均不显示假数据。
5. 工具不获取键盘焦点，不修改 Codex，不读取聊天内容和账号令牌。
6. 自动化测试通过，并完成真实 Codex 窗口手动验收。
7. 安装包、便携包、校验值、许可证和中英文基础文档齐全。

## 13. 已知风险与控制

- App Server 协议随 Codex 版本演进：使用宽容读取、显式版本检查、协议契约测试和清晰更新提示。
- Windows 窗口事件可能丢失：事件驱动为主，低频位置校准为辅。
- 最大化无外侧空间：使用已确认的 4px 标题栏内侧降级模式。
- 多显示器 DPI 可能造成抖动：所有坐标先归一到目标显示器 DPI，再一次性提交位置。
- 无代码签名证书会触发 SmartScreen：发布说明明确风险，CI 预留签名步骤，正式推广前配置受信任签名。
- Codex 视觉更新可能改变标题栏安全区域：最大化模式使用保守边距，并把位置规则集中在可更新策略中。
