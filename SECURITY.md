# 安全策略

## 支持范围

安全修复优先覆盖最新稳定版本。发布前候选版本不承诺长期维护。

## 报告漏洞

请不要公开披露尚未修复、可被利用的漏洞。优先通过 GitHub 的 **Report a vulnerability** 私密报告，说明受影响版本、复现步骤、影响和建议修复方式。不要附带 Codex 令牌、聊天、邮箱、账号 ID 或其他个人数据。

如果仓库尚未启用私密漏洞报告，请仅创建不包含利用细节的普通 Issue，等待维护者提供安全联络方式。

## 安全边界

- 本项目只连接本机 Codex App Server 的标准输入输出，不提供网络服务端口。
- 不读取或存储账号令牌、聊天内容、项目内容和剪贴板。
- 手动更新检查只访问 `api.github.com/repos/lingge66/codex-quota-rail/releases/latest`。
- 发布前请核对 SHA-256 和 SPDX SBOM；未签名构建会明确标注。
